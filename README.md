# Трекер прочитанных книг

Учебный консольный проект на Python для практики командной разработки с использованием Git и GitHub.

## Назначение приложения

Программа ведет список прочитанных книг и поддерживает:

- добавление книги;
- просмотр всех книг;
- расчет средней оценки;
- статистику по авторам;
- удаление книги.

Данные сохраняются в `books.json`.

## Структура проекта

- `main.py` — консольное меню и обработка действий пользователя;
- `tracker.py` — бизнес-логика и работа с данными;
- `books.json` — JSON-хранилище книг;
- `tests/test_tracker.py` — базовые модульные тесты.

## Полный сценарий выполнения ТЗ (Windows, cmd)

Ниже приведен пошаговый порядок от нуля до финальной сдачи.

## 1) Установка инструментов

### Python

1. Откройте https://www.python.org/downloads/
2. Скачайте последнюю версию Python 3.
3. При установке обязательно отметьте **Add Python to PATH**.
4. Проверьте:

```bash
python --version
```

### Git

1. Откройте https://git-scm.com/download/win
2. Скачайте и установите Git (настройки по умолчанию).
3. Проверьте:

```bash
git --version
```

## 2) Создание проекта и инициализация Git

```bash
cd /d C:\Users\mihad\Desktop
mkdir TZ-GIT
cd TZ-GIT
git init
git branch -M main
git config user.name "kappaberry"
git config user.email "ainswg08@gmail.com"
```

## 3) Первый коммит (обязательный по ТЗ)

Создайте файлы:

- `README.md`
- `.gitignore`
- `books.json` (содержимое `[]`)

Пример `.gitignore`:

```text
__pycache__/
*.pyc
.venv/
venv/
env/
```

Команды:

```bash
git add README.md .gitignore books.json
git commit -m "Начальная структура проекта"
```

## 4) Создание GitHub-репозитория и подключение origin

1. На GitHub нажмите **New repository**.
2. Тип репозитория: **Public**.
3. Подключите удаленный репозиторий:

```bash
git remote add origin https://github.com/<USERNAME>/<REPO>.git
git push -u origin main
```

Проверка:

```bash
git remote -v
```

## 5) Создание Issue (обязательный пункт)

На GitHub:

- откройте `Issues` -> `New issue`;
- вставьте текст ниже.

### Готовый шаблон Issue

**Title**:

```text
Добавить проверку на дубликаты книг
```

**Description**:

```text
Пользователь не должен добавлять одну и ту же книгу дважды.
Критерий: сравнение по паре "автор + название".
Ожидаемое поведение: при попытке повторного добавления приложение выводит сообщение о дубликате и не изменяет books.json.
```

## 6) Ветка feature/add-book

```bash
git checkout -b feature/add-book
```

Реализуйте в коде:

- добавление книги;
- валидацию оценки (1..5);
- проверку формата даты;
- проверку дубликатов по `автор + название`.

Команды:

```bash
git add main.py tracker.py
git commit -m "Добавить функционал добавления книги и проверку дубликатов. Closes #1"
git push -u origin feature/add-book
```

## 7) Pull Request для feature/add-book (обязательный пункт)

На GitHub:

1. `Pull requests` -> `New pull request`
2. `base: main`, `compare: feature/add-book`
3. Создайте PR, используя текст ниже
4. Нажмите `Merge pull request`
5. Нажмите `Delete branch`

### Готовый текст PR

**Title**:

```text
Добавлен функционал добавления книг
```

**Description**:

```text
## Что сделано
- Реализован пункт меню "Добавить книгу"
- Добавлена валидация оценки (1-5)
- Добавлена валидация даты прочтения (YYYY-MM-DD)
- Добавлена проверка на дубликаты по паре автор + название

## Связанные задачи
Closes #1

## Проверка
- Добавление новой книги работает
- Повторное добавление той же книги блокируется
```

## 8) Ветка feature/list-and-stats

```bash

git pull
git checkout -b feature/list-and-stats
```

Реализуйте:

- пункт 2: вывод всех книг;
- пункт 3: средняя оценка;
- пункт 4: статистика по авторам.

Команды:

```bash

git commit -m "Добавить вывод списка книг, среднюю оценку и статистику по авторам"

```

## 9) Ветка feature/delete

```bash
git checkout main
git pull
git checkout -b feature/delete
```

Реализуйте:

- удаление книги по индексу;
- удаление по паре автор + название.

Команды:

```bash

git commit -m "Добавить удаление книги по индексу и по автору с названием"

```

## 10) Слияние оставшихся веток в main

Вариант из методички — локальное слияние:


## 13) Что сдавать преподавателю

1. Ссылка на публичный GitHub-репозиторий.
2. Скрин графа коммитов:

```bash

```

3. Скрин вкладки `Pull Requests` (виден закрытый/merged PR).
4. Скрин вкладки `Issues` -> `Closed` (виден закрытый Issue).

## 14) Частые ошибки и решения

- `fatal: pathspec ... did not match any files`  
  Вы не в папке проекта. Перейдите в нужную директорию:

```bash
cd /d C:\Users\mihad\Desktop\TZ-GIT
```

- `Author identity unknown`  
  Не настроен пользователь Git:

```bash
git config user.name "kappaberry"
git config user.email "ainswg08@gmail.com"
```

- `origin does not appear to be a git repository`  
  Не подключен удаленный репозиторий или неверный URL:

```bash
git remote add origin https://github.com/<USERNAME>/<REPO>.git
```

или

```bash
git remote set-url origin https://github.com/<USERNAME>/<REPO>.git
```
# Трекер прочитанных книг

Учебный консольный проект на Python для практики командной разработки с использованием Git и GitHub.

## Назначение приложения

Программа помогает вести список прочитанных книг и поддерживает:

- добавление книги (автор, название, оценка, дата прочтения);
- просмотр всех книг;
- расчёт средней оценки;
- статистику по авторам;
- удаление книги.

Данные сохраняются в файле `books.json`.

## Структура проекта

- `main.py` — интерфейс консольного меню и обработка действий пользователя;
- `tracker.py` — бизнес-логика и работа с данными;
- `books.json` — хранилище книг в формате JSON;
- `tests/test_tracker.py` — базовые модульные тесты.

## Установка и подготовка (Windows)

### 1. Установите Python

1. Скачайте Python с [python.org](https://www.python.org/downloads/).
2. При установке включите опцию **Add Python to PATH**.
3. Проверьте:

```bash
python --version
```

### 2. Установите Git

1. Скачайте Git с [git-scm.com](https://git-scm.com/download/win).
2. Установите с настройками по умолчанию.
3. Проверьте:

```bash
git --version
```

### 3. Клонирование или запуск из текущей папки

Если репозиторий уже на компьютере, перейдите в папку проекта:

```bash
cd /d C:\Users\"Ваше Имя Пользователя"\Desktop\TZ-GIT
```

## Запуск приложения

```bash
python main.py
```

После запуска появится меню:

```text
Трекер прочитанных книг
1. Добавить книгу
2. Показать все книги
3. Показать среднюю оценку
4. Статистика по авторам
5. Удалить книгу
6. Выход
```

## Использование пунктов меню

### 1. Добавить книгу

Введите:

- автора;
- название;
- оценку (целое число от 1 до 5);
- дату прочтения в формате `YYYY-MM-DD`.

Если такая книга (автор + название) уже существует, будет выведено предупреждение о дубликате.

### 2. Показать все книги

Выводится список с индексами, автором, названием, оценкой и датой.

### 3. Показать среднюю оценку

Показывается среднее арифметическое всех оценок.

### 4. Статистика по авторам

Показывается количество книг по каждому автору.

### 5. Удалить книгу

Поддерживаются два способа удаления:

- по индексу из списка;
- по паре автор + название.

### 6. Выход

Завершает работу программы.
